## 项目简介

奶龙量化是一套以 Streamlit 构建的 A 股量化研究与交易验证平台。项目将数据获取、指标计算、策略回测、公式 Alpha 研究、组合分析、自动选股、模拟交易和自动化执行整合在统一界面中，并通过机构研究风格的可视化呈现关键结论。

本仓库是**本地完整版**，强调研究功能完整性与本机运行体验；它不包含后续添加的 Docker、云服务器或临时公网隧道配置。

## 核心能力

- **策略验证**：支持单标的历史回测、复权方式、手续费、滑点和交易规则配置。
- **Alpha² 实验室**：构造量纲合理、可解释的公式 Alpha，按训练集、验证集和测试集隔离研究，报告 IC、Rank IC、相关性与样本外表现。
- **组合归因**：对多标的组合进行统一回测，比较收益、回撤、交易成本与个股贡献。
- **信号筛选**：扫描 A 股候选池，按技术条件和综合评分筛选标的，并与后续执行模块联动。
- **模拟执行**：使用本地持久化账户验证现金、仓位、订单和交易记录，不承担真实资金风险。
- **自动交易框架**：提供策略轮询、仓位约束和执行日志；Windows 环境可选配 `easytrader` 适配层。
- **研究工作台**：持久保存关注池、策略快照与追加式审计记录，方便复盘研究过程。
- **真实数据链路**：支持 `efinance`、AKShare、BaoStock 及平台历史源；系统不会用随机模拟行情冒充真实回测数据。

## 研究流程

```text
真实行情 → 信号筛选 → 单标的验证 → Alpha² 研究 → 组合归因 → 模拟执行 → 审计复盘
```

推荐先在自动选股模块形成候选池，再依次完成单标的和组合回测。只有在成本、回撤及样本外表现均可接受后，才进入模拟执行阶段。

## 快速开始

### 环境要求

- Python 3.10 或更高版本
- macOS、Windows 或 Linux
- 能访问所选行情数据源的网络环境

### 安装与运行

```bash
git clone https://github.com/493581831-hue/nailong-quant.git
cd nailong-quant

python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
python3 -m pip install -r requirements.txt
python3 -m streamlit run app.py
```

启动后浏览器通常会自动打开 `http://localhost:8501`。

## 数据说明

| 场景 | 数据能力 | 说明 |
|---|---|---|
| 历史回测 | efinance / 平台历史源 / BaoStock | 依模块提供前复权、后复权或不复权选项 |
| 实时行情 | AKShare 及公开行情接口 | 上游不可用时会明确提示，不生成虚假实时数据 |
| Alpha² | 单标的 OHLCV 历史数据 | 采用严格时间切分与下一交易日执行规则 |
| 模拟账户 | 本地 JSON 状态 | 默认只保存在本机，不应提交到 Git 仓库 |

公开行情接口可能因网络、频率限制或上游变更而暂时不可用。若某个数据源失败，请稍后重试或在界面中切换数据源。

## Alpha² 方法边界

本项目中的 Alpha² 模块是面向单资产时序研究的工程适配：保留量纲检查、未来 20 日目标、低相关选择、组合信号及样本外回测等核心思想，但不是原论文横截面 DRL + MCTS 基准的逐行复刻，也不应与论文报告结果直接比较。

为降低数据泄漏风险，模型选择只使用训练集和验证集信息，测试集仅用于最终评估；交易信号在形成后的下一交易日执行，并按设置扣除换手成本。

## 本地数据与隐私

运行后产生的模拟账户、关注池、策略快照、审计日志、缓存和环境变量已由 `.gitignore` 排除。提交代码前仍建议检查工作区，避免将券商配置、账号信息或个人研究数据加入版本控制。

## 已知限制

- 公开数据源的稳定性和字段口径不由本项目控制。
- 回测尚不能完整复现实际成交中的冲击成本、流动性约束、停牌和极端行情。
- `easytrader` 实盘适配仅支持 Windows，并要求用户自行配置已登录的交易客户端。
- 自动交易属于实验性能力，建议始终先使用模拟盘并设置严格的仓位上限。

## 风险声明

本项目仅用于学习、研究与流程验证，不构成任何投资建议、收益承诺或交易依据。历史回测表现不代表未来收益；数据错误、模型失效、交易成本、流动性和执行偏差均可能造成重大损失。任何实盘决策及其后果均由使用者自行承担。
